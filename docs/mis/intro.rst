.. _mjs-intro:

Написання програм на mJS
========================

Keira має вбудовану підтримку мови програмування **mJS** — легковагового JavaScript-інтерпретатора,
оптимізованого для мікроконтролерів. Це дозволяє писати та виконувати програми на JavaScript
прямо з SD-картки, без необхідності компіляції чи перепрошивання Лілки.

.. contents::
    :local:
    :depth: 1

.. _mjs-vs-js:

Що таке mJS — і чим він відрізняється від JavaScript?
------------------------------------------------------

Якщо ви чули про **JavaScript** — мову, яку використовують у веббраузерах і на серверах — то
**mJS** є її «легкою» версією, спеціально створеною для мікроконтролерів з дуже обмеженою пам'яттю
(таких як ESP32 у Лілці).

Уявіть різницю так:

- Звичайний JavaScript — це повноцінний двигун автомобіля з усіма системами.
- mJS — це компактний мотоциклетний двигун: він робить те саме головне, але займає набагато менше місця.

.. list-table::
   :header-rows: 1
   :widths: 35 32 33

   * -
     - JavaScript (браузер / Node.js)
     - mJS (Лілка)
   * - Синтаксис
     - Повний стандарт ES6+
     - Підмножина ES6 (базовий синтаксис)
   * - Потрібна пам'ять
     - Мегабайти
     - Кілобайти (підходить для ESP32)
   * - Де виконується
     - Комп'ютер, сервер, браузер
     - Прямо на Лілці
   * - Класи, Promise, async/await
     - Так
     - Ні (не підтримується)
   * - Змінні ``let`` / ``var``
     - Так
     - Так
   * - Функції ``function`` / ``() =>``
     - Так
     - Так
   * - Цикли ``while`` / ``for``
     - Так
     - Так
   * - Умови ``if`` / ``else``
     - Так
     - Так

.. note::

    Якщо ви раніше писали JavaScript — mJS здасться вам знайомим. Якщо ні — не хвилюйтесь:
    для роботи з GPIO потрібно знати лише кілька простих конструкцій, і всі вони показані
    в прикладах нижче.

.. warning::

    Якщо ви зовсім не знайомі з JavaScript, рекомендуємо переглянути короткий вступ перед тим,
    як продовжувати: https://javascript.info/first-steps

.. _mjs-how:

Як mJS потрапляє на Лілку?
--------------------------

Ви пишете код у звичайному текстовому файлі з розширенням ``.js`` на комп'ютері, копіюєте
його на SD-картку та вставляєте картку в Лілку. Далі в меню Лілки обираєте цей файл —
і він одразу виконується. Ніякої компіляції, ніяких кабелів.

.. _mjs-example:

Приклад програми
----------------

mJS сам по собі — це мова програмування загального призначення, яка не має вбудованих функцій
для роботи з GPIO тощо. Для роботи з апаратним забезпеченням Лілки в mJS доступні спеціальні
вбудовані об'єкти.

Об'єкт ``gpio`` завантажується автоматично — не потрібно писати жодних ``require(...)``.
Ось приклад простої програми на mJS, яка вмикає світлодіод на виводі 25:

.. code-block:: javascript
    :linenos:

    // Налаштовуємо вивід 25 як OUTPUT:
    gpio.set_mode(25, gpio.OUTPUT);

    // Встановлюємо HIGH — вмикаємо світлодіод:
    gpio.write(25, gpio.HIGH);

Ви можете зберегти цей код у файл з розширенням ``.js`` на SD-картці, а потім виконати його,
обравши його в браузері SD-картки.

.. _mjs-example-button:

Читання стану кнопки
~~~~~~~~~~~~~~~~~~~~

Ось більш розгорнутий приклад — зчитування значення кнопки, підключеної до виводу 34:

.. code-block:: javascript
    :linenos:

    // Налаштовуємо вивід 34 як INPUT з підтягуючим резистором:
    gpio.set_mode(34, gpio.INPUT_PULLUP);

    // Читаємо поточний стан:
    let val = gpio.read(34);

    if (val === gpio.LOW) {
        print("Кнопка натиснута!\n");
    } else {
        print("Кнопка відпущена.\n");
    }

.. note::

    У цьому прикладі кнопка підключена між виводом 34 та GND. Завдяки ``gpio.INPUT_PULLUP``,
    вивід підтягнутий до ``HIGH``, і при натисканні кнопки рівень падає до ``LOW``.

.. _mjs-example-analog:

Читання аналогового сигналу
~~~~~~~~~~~~~~~~~~~~~~~~~~~

Для зчитування аналогового сигналу (наприклад, з потенціометра або датчика) використовується
функція ``gpio.analog_read()``, яка повертає значення від 0 до 4095:

.. code-block:: javascript
    :linenos:

    // Налаштовуємо аналоговий вхід на виводі 36:
    gpio.set_mode(36, gpio.INPUT);

    // Зчитуємо аналогове значення (0..4095):
    let analog = gpio.analog_read(36);

    print("Аналогове значення: ", analog, "\n");

.. warning::

    Не всі виводи мікроконтролера підтримують аналоговий вхід. Зверніться до схеми Лілки,
    щоб визначити, які виводи мають функцію АЦП.

.. _mjs-gpio:

``gpio`` — Керування GPIO-пінами
---------------------------------

Об'єкт ``gpio`` є глобальним і доступний у будь-якому mJS-скрипті без додаткових оголошень.
Він надає методи для роботи з цифровими та аналоговими виводами мікроконтролера, а також
набір зручних констант.

.. note::

    Назви методів та констант об'єкта ``gpio`` відповідають стандартним функціям фреймворку
    Arduino: ``pinMode()``, ``digitalWrite()``, ``digitalRead()``, ``analogRead()``,
    ``HIGH``, ``LOW``, ``INPUT``, ``OUTPUT`` тощо.

.. _mjs-gpio-methods:

Методи
~~~~~~

``gpio.set_mode(pin, mode)``
^^^^^^^^^^^^^^^^^^^^^^^^^^^^

Налаштовує режим роботи виводу мікроконтролера.

:param pin: Номер виводу (GPIO pin number).
:type pin: number
:param mode: Режим роботи: ``gpio.INPUT``, ``gpio.OUTPUT``, ``gpio.INPUT_PULLUP`` або ``gpio.INPUT_PULLDOWN``.
:type mode: number

.. code-block:: javascript
    :linenos:

    gpio.set_mode(25, gpio.OUTPUT);       // вивід 25 — вихід
    gpio.set_mode(34, gpio.INPUT_PULLUP); // вивід 34 — вхід з підтяжкою

``gpio.write(pin, value)``
^^^^^^^^^^^^^^^^^^^^^^^^^^

Встановлює цифровий рівень на виводі. Вивід має бути попередньо налаштований як ``OUTPUT``
за допомогою ``gpio.set_mode()``.

:param pin: Номер виводу.
:type pin: number
:param value: Значення: ``gpio.HIGH`` (1) або ``gpio.LOW`` (0).
:type value: number

.. code-block:: javascript
    :linenos:

    gpio.set_mode(25, gpio.OUTPUT);
    gpio.write(25, gpio.HIGH); // вмикаємо
    gpio.write(25, gpio.LOW);  // вимикаємо

``gpio.read(pin)``
^^^^^^^^^^^^^^^^^^

Зчитує цифровий рівень виводу. Вивід має бути попередньо налаштований як ``INPUT``,
``INPUT_PULLUP`` або ``INPUT_PULLDOWN``.

:param pin: Номер виводу.
:type pin: number
:returns: ``gpio.HIGH`` (1) або ``gpio.LOW`` (0).
:rtype: number

.. code-block:: javascript
    :linenos:

    gpio.set_mode(34, gpio.INPUT_PULLUP);
    let val = gpio.read(34);

    if (val === gpio.LOW) {
        print("Кнопка натиснута\n");
    }

``gpio.analog_read(pin)``
^^^^^^^^^^^^^^^^^^^^^^^^^

Зчитує аналогове значення виводу за допомогою АЦП (аналогово-цифрового перетворювача)
мікроконтролера. Вивід має бути налаштований як ``gpio.INPUT``.

:param pin: Номер виводу (має підтримувати аналоговий вхід).
:type pin: number
:returns: Ціле число від 0 до 4095 (12-бітне значення АЦП).
:rtype: number

.. code-block:: javascript
    :linenos:

    gpio.set_mode(36, gpio.INPUT);
    let brightness = gpio.analog_read(36);
    // brightness: 0 = мінімум, 4095 = максимум

.. _mjs-gpio-constants:

Константи
~~~~~~~~~

Об'єкт ``gpio`` надає набір зручних числових констант. Їх рекомендовано використовувати
замість «магічних чисел» для кращої читабельності коду.

.. list-table::
   :header-rows: 1
   :widths: 30 15 55

   * - Константа
     - Значення
     - Опис
   * - ``gpio.LOW``
     - 0
     - Цифровий рівень LOW (логічний 0)
   * - ``gpio.HIGH``
     - 1
     - Цифровий рівень HIGH (логічна 1)
   * - ``gpio.INPUT``
     - INPUT
     - Режим цифрового входу
   * - ``gpio.OUTPUT``
     - OUTPUT
     - Режим цифрового виходу
   * - ``gpio.INPUT_PULLUP``
     - INPUT_PULLUP
     - Вхід з підтягуючим резистором (pull-up)
   * - ``gpio.INPUT_PULLDOWN``
     - INPUT_PULLDOWN
     - Вхід зі стягуючим резистором (pull-down)

``gpio.LOW`` / ``gpio.HIGH``
''''''''''''''''''''''''''''

Цифрові рівні «низький» (0) та «високий» (1). Використовуйте їх замість чисел 0 і 1:

.. code-block:: javascript
    :linenos:

    gpio.write(25, gpio.HIGH); // замість gpio.write(25, 1)
    gpio.write(25, gpio.LOW);  // замість gpio.write(25, 0)

``gpio.INPUT``
''''''''''''''

Режим виводу «цифровий вхід». Вивід зчитує сигнал ззовні. Підтягуючий або стягуючий
резистор не активується.

``gpio.OUTPUT``
'''''''''''''''

Режим виводу «цифровий вихід». Вивід керує рівнем напруги (``HIGH`` або ``LOW``).

``gpio.INPUT_PULLUP``
'''''''''''''''''''''

Режим «цифровий вхід з внутрішнім підтягуючим резистором» (pull-up). Вивід підтягується
до рівня ``HIGH`` через внутрішній резистор. Це зручно для підключення кнопок без зовнішніх
резисторів — кнопка замикає вивід на GND, і рівень падає до ``gpio.LOW``.

``gpio.INPUT_PULLDOWN``
'''''''''''''''''''''''

Режим «цифровий вхід з внутрішнім стягуючим резистором» (pull-down). Вивід стягується
до рівня ``LOW`` через внутрішній резистор. Рівень піднімається до ``gpio.HIGH`` лише
при подачі зовнішнього сигналу.

.. _mjs-example-blink:

Повний приклад: мигання світлодіода
------------------------------------

Наступний приклад демонструє мигання світлодіода на виводі 25:

.. code-block:: javascript
    :linenos:

    // Налаштовуємо вивід 25 як вихід:
    gpio.set_mode(25, gpio.OUTPUT);

    // Мигаємо 10 разів:
    let i = 0;
    while (i < 10) {
        gpio.write(25, gpio.HIGH); // вмикаємо

        let j = 0;
        while (j < 500000) { j++; } // затримка ~0.5 с

        gpio.write(25, gpio.LOW);  // вимикаємо

        j = 0;
        while (j < 500000) { j++; } // затримка ~0.5 с

        i++;
    }

    print("Готово!\n");

.. warning::

    mJS не має вбудованої функції ``delay()``. Для затримок використовуйте зайнятий цикл,
    як показано вище, або перевірте доступні утиліти у вашій версії Keira.

.. _mjs-example-button-led:

Повний приклад: кнопка та LED
------------------------------

Цей приклад показує, як поєднати читання стану кнопки (вивід 34) з керуванням світлодіодом
(вивід 25). Поки кнопка натиснута — LED горить:

.. code-block:: javascript
    :linenos:

    // Налаштовуємо виводи:
    gpio.set_mode(25, gpio.OUTPUT);       // LED — вихід
    gpio.set_mode(34, gpio.INPUT_PULLUP); // кнопка — вхід з підтяжкою

    // Нескінченний цикл опитування кнопки:
    while (true) {
        let btn = gpio.read(34);

        if (btn === gpio.LOW) {
            // Кнопка натиснута (замикає на GND):
            gpio.write(25, gpio.HIGH);
            print("Кнопка натиснута — LED увімкнено\n");
        } else {
            // Кнопка відпущена:
            gpio.write(25, gpio.LOW);
        }
    }

.. note::

    У цьому прикладі кнопка підключена між виводом 34 та GND. Завдяки ``gpio.INPUT_PULLUP``,
    вивід підтягнутий до ``HIGH``, і при натисканні кнопки рівень падає до ``LOW``.

Повний приклад: читання аналогового датчика
--------------------------------------------

Цей приклад зчитує 5 значень з потенціометра або іншого аналогового датчика і виводить їх
через ``print``:

.. code-block:: javascript
    :linenos:

    // Налаштовуємо аналоговий вхід:
    gpio.set_mode(36, gpio.INPUT);

    // Зчитуємо 5 значень:
    let i = 0;
    while (i < 5) {
        let val = gpio.analog_read(36);
        print("Зчитане значення: ", val, "\n");

        let j = 0;
        while (j < 200000) { j++; } // коротка затримка

        i++;
    }

.. _mjs-example-traffic-light:

Повний приклад: «Світлофор»
-----------------------------

Програма, яка імітує світлофор з трьома світлодіодами: червоний на виводі 25, жовтий на 26,
зелений на 27. Послідовність: зелений (3 с) → жовтий (1 с) → червоний (3 с) → і знову.

Спочатку запишемо номери виводів у змінні, щоб не плутатись у коді. Так набагато зручніше,
ніж писати скрізь цифри 25, 26, 27:

.. code-block:: javascript
    :linenos:

    let PIN_RED    = 25;
    let PIN_YELLOW = 26;
    let PIN_GREEN  = 27;

mJS не має вбудованої функції ``delay()``, тому зробимо власну просту затримку через зайнятий
цикл. Функція ``wait(n)`` просто «крутиться» у циклі, витрачаючи час:

.. code-block:: javascript
    :linenos:

    // wait(1) — приблизно 1 секунда
    function wait(seconds) {
        let i = 0;
        while (i < seconds * 800000) { i++; }
    }

.. warning::

    Точність такої затримки невисока — вона залежить від завантаженості процесора.
    Для точних затримок рекомендуємо використовувати апаратні таймери, якщо вони
    доступні у вашій версії Keira.

Повний код (збережіть у файл ``traffic_light.js``):

.. code-block:: javascript
    :linenos:

    // ===== Світлофор на mJS =====

    let PIN_RED    = 25;
    let PIN_YELLOW = 26;
    let PIN_GREEN  = 27;

    // wait(1) — приблизно 1 секунда
    function wait(seconds) {
        let i = 0;
        while (i < seconds * 800000) { i++; }
    }

    // Налаштовуємо виводи як виходи та вимикаємо їх:
    gpio.set_mode(PIN_RED,    gpio.OUTPUT);
    gpio.set_mode(PIN_YELLOW, gpio.OUTPUT);
    gpio.set_mode(PIN_GREEN,  gpio.OUTPUT);

    gpio.write(PIN_RED,    gpio.LOW);
    gpio.write(PIN_YELLOW, gpio.LOW);
    gpio.write(PIN_GREEN,  gpio.LOW);

    // Головний цикл — світлофор працює постійно:
    while (true) {
        // --- ЗЕЛЕНИЙ: дозволено їхати ---
        gpio.write(PIN_GREEN,  gpio.HIGH);
        gpio.write(PIN_YELLOW, gpio.LOW);
        gpio.write(PIN_RED,    gpio.LOW);
        wait(3);

        // --- ЖОВТИЙ: увага ---
        gpio.write(PIN_GREEN,  gpio.LOW);
        gpio.write(PIN_YELLOW, gpio.HIGH);
        gpio.write(PIN_RED,    gpio.LOW);
        wait(1);

        // --- ЧЕРВОНИЙ: стоп ---
        gpio.write(PIN_GREEN,  gpio.LOW);
        gpio.write(PIN_YELLOW, gpio.LOW);
        gpio.write(PIN_RED,    gpio.HIGH);
        wait(3);
    }
